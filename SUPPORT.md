# Support

Use GitHub Discussions for usage questions, integration help, and design questions:

https://github.com/oaslananka/health-monitor-mcp/discussions

Open a GitHub issue only for tracked work: reproducible bugs, release or CI blockers, feature
requests, and documentation or governance gaps. Use the issue forms so reports include enough
context to reproduce or validate the work.

Security vulnerabilities must not be reported in public issues or discussions. Follow
[SECURITY.md](SECURITY.md) and use the repository security advisory workflow.

## Response Targets

Response targets begin when an issue or discussion contains enough information for a maintainer
to act.

| Channel                                           | Target                              |
| ------------------------------------------------- | ----------------------------------- |
| P0 security, release, CI, or package blocker      | 1 business day                      |
| P1 compatibility, product, or governance issue    | 3 business days                     |
| P2 quality, testing, DX, or maintainability issue | 7 business days                     |
| P3 polish, demo, community, or roadmap request    | Best effort during backlog review   |
| Discussions support                               | Best effort; issues take precedence |

## Scope

Maintainers support the current documented release line, the Node versions declared in
`package.json`, and the public workflows documented in README.md and docs/. Unsupported forks,
private deployments, and local environment customizations may be redirected to Discussions.
